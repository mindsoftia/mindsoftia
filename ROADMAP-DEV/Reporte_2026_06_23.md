# Reporte de Desarrollo - 23 de Junio de 2026

**Proyecto:** Mindsoftia (SaaS B2B)
**Módulos Afectados:** Autenticación, Base de Datos (Supabase), Frontend (React/Vite), Backend (Laravel).

## Resumen de Actividades

En la jornada de hoy se logró consolidar la arquitectura core del modelo multi-tenant (Multi-empresa) para garantizar la separación de entornos y reforzar la seguridad general del ecosistema, además de crear el centro de control del Superadmin.

### 1. Arquitectura Multi-Tenant y Seguridad de Storage (Supabase)
- **Implementación de RLS (Row Level Security):** Se generó un script SQL maestro para Supabase.
- **Función `current_tenant_id()`:** Se creó una función nativa en PostgreSQL que lee el `tenant_id` directamente del token JWT (`app_metadata`) del usuario en tiempo real.
- **Aislamiento de Archivos:** Se configuraron políticas en Supabase Storage (Bucket: `archivos_empresas`) que impiden tajantemente que una empresa pueda subir, ver o eliminar archivos de la carpeta de otra empresa.

### 2. Integración Laravel + Supabase Auth
- **Inyección de Identidad (JWT):** Se modificó el `OnboardingController.php` (Laravel). Tras crear el registro de la nueva empresa en la DB, el servidor usa la clave `SERVICE_ROLE` para comunicarse con la **API Admin de Supabase** e inyectarle el `tenant_id` al perfil del usuario.

### 3. Finalización del Onboarding (Frontend)
- Se desarrolló el `StorageService` en React.
- **Actualización del `OnboardingWizard.jsx`:**
  - Se añadió la capacidad de capturar el logotipo corporativo en el Paso 1.
  - Se implementó el comando `supabase.auth.refreshSession()` justo al finalizar la petición a Laravel para forzar la actualización del token.
  - Se configuró la subida automática y segura del logotipo a la carpeta exclusiva de la empresa recién creada.

### 4. Seguridad de Sesión Nivel Bancario (PWA)
- **Persistencia Volátil:** Se migró el cliente global de Supabase (`supabase.js`) de usar `localStorage` a `sessionStorage`. Al cerrar la PWA o pestaña, la sesión se destruye.
- **Auto-Logout (Inactividad):** Se implementó un detector global de eventos en `App.jsx` que desconecta automáticamente al usuario tras 15 minutos sin interacción (mouse, teclado o táctil), redirigiéndolo al Login.

### 5. Centro de Control SaaS (Superadmin Dashboard)
- Se reescribió completamente `src/views/Dashboard.jsx` para dejar de ser una plantilla y convertirse en el monitor principal del dueño del software.
- **Métricas Agregadas:** MRR (Ingreso Mensual Recurrente), ARPU, Tasa de Churn, y Empresas Activas.
- **Gráficos de Telemetría:** Un gráfico azul principal de crecimiento MRR y un Doughnut Chart para el "Uso de Módulos".
- **Monitor DevOps:** Tarjetas en tiempo real para visualizar el límite de conexiones DB de PostgreSQL, cuota de disco duro de Storage (1.2 TB simulación) y salud de la API DIAN.
- **Auditoría:** Se incluyó un "Activity Feed" lateral simulado para trackear en tiempo real cambios de planes y errores del sistema.

### 6. Control de Versiones
- Se empaquetó todo el trabajo con `git add .`, se documentó con un commit semántico y se sincronizó (`git push`) al repositorio `origin/main` en GitHub exitosamente.
