# Mindsoftia - Roadmap & Changelog Diario

## 📅 Fecha: 13 de Julio de 2026

### 🛡️ Fase 1: Arquitectura de Seguridad (Multi-Tenant & RBAC)
1. **Blindaje de Rutas React (`ProtectedRoute.jsx`):**
   - Se migró de una protección básica a una validación estricta de **Módulos** (`hasModule`) y **Permisos** (`hasPermission`).
   - Se solucionó un bug del enrutador de React (v6) añadiendo el `<Outlet />` faltante que causaba que las pantallas (como el POS) se renderizaran en blanco al ser protegidas.
2. **Master Key para Pruebas:**
   - Se configuró temporalmente el rol de `'propietario'` como llave maestra en `authStore.js` y `AuthController.php`, bypasseando el bloqueo automático provocado por la desactivación de módulos por defecto en la base de datos local.
3. **Migración de Base de Datos (Backend Laravel):**
   - Se ejecutaron migraciones en PostgreSQL para incluir columnas booleanas de los módulos `modulo_compras` y `modulo_contabilidad` en la tabla `empresas`, sincronizando el acceso SaaS con el frontend.

### 🎨 Fase 2: UI/UX & Mejoras de Experiencia (Master-UI)
1. **Refactorización Vista Proveedores:**
   - Se removió el botón tradicional "+ Nuevo Proveedor".
   - Se implementó un moderno **Botón Flotante (FAB)** circular en la esquina inferior derecha con efecto hover, mejorando la usabilidad móvil y liberando espacio en las cabeceras de tabla.

### 🛒 Fase 3: Activación del Punto de Venta (POS)
1. **Mapeo de Datos Base de Datos ↔ Frontend:**
   - Se detectó un fallo crítico donde los productos se renderizaban como deshabilitados. Se corrigió el mapeo de variables:
     - `activo` ➡️ `estado`
     - `precio_venta` ➡️ `precio_venta_1`
     - `referencia` ➡️ `codigo_sku`
2. **Selector de Clientes en Carrito (Facturación Electrónica):**
   - Se creó el componente `ClienteSelector.jsx`.
   - Búsqueda en tiempo real (IndexedDB) para anexar el ID y Nombre del cliente a la venta (vital para futura DIAN).
3. **Cálculos Contables y Tributarios:**
   - El carrito (`CarritoPanel.jsx` y `POSLayout.jsx`) se modificó para abandonar los subtotales brutos.
   - Ahora calcula automáticamente el **Subtotal Gravable**, deduce el **IVA (19% por defecto)** y entrega el **Total a Pagar**, asegurando concordancia con los estándares contables requeridos.
